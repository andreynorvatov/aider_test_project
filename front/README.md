# Frontend Project

Современная структура фронтенд-проекта.

## Структура проекта

```
front/
├── public/              # Статические файлы
│   ├── index.html       # Главный HTML-файл
│   ├── favicon.ico      # Иконка сайта
│   └── manifest.json    # PWA манифест
│
├── src/                 # Исходный код
│   ├── assets/          # Ресурсы (изображения, шрифты, иконки)
│   │   ├── fonts/       # Шрифты
│   │   ├── icons/       # SVG иконки
│   │   └── images/      # Изображения
│   │
│   ├── components/      # React/Vue компоненты
│   │   ├── common/      # Общие UI компоненты (кнопки, инпуты)
│   │   ├── layout/      # Компоненты layout (Header, Footer, Sidebar)
│   │   └── features/    # Функциональные компоненты
│   │
│   ├── config/          # Конфигурация приложения
│   ├── constants/       # Константы и неизменяемые значения
│   ├── hooks/           # Кастомные React хуки / Vue composables
│   ├── pages/           # Страницы (роуты)
│   ├── router/          # Настройка маршрутизации
│   ├── services/        # API сервисы и внешние интеграции
│   │   └── api/         # API клиент и эндпоинты
│   ├── store/           # Управление состоянием (Redux/Zustand/Vuex)
│   │   └── slices/      # Redux slices / stores
│   ├── styles/          # Глобальные стили
│   │   ├── base/        # Базовые стили (reset, переменные)
│   │   ├── components/  # Стили компонентов
│   │   └── utils/       # SCSS миксины и функции
│   ├── types/           # TypeScript типы
│   ├── utils/           # Вспомогательные функции
│   └── __tests__/       # Тесты
│
├── .env                 # Переменные окружения
├── .env.example         # Пример переменных окружения
├── package.json         # Зависимости и скрипты
├── vite.config.js       # Конфигурация Vite
├── tsconfig.json        # Конфигурация TypeScript
└── eslint.config.js     # Конфигурация ESLint
```

## Начало работы

```bash
# Установка зависимостей
npm install

# Запуск в режиме разработки
npm run dev

# Сборка для продакшена
npm run build

# Запуск тестов
npm run test

# Линтинг
npm run lint
```

## Рекомендации

1. **Компоненты**: Размещайте переиспользуемые UI компоненты в `components/common/`
2. **Стили**: Используйте CSS-модули или styled-components для изоляции стилей
3. **API**: Все API вызовы должны быть в `services/api/`
4. **Состояние**: Используйте store только для глобального состояния
5. **Типы**: Держите типы в `types/` для общих, рядом с компонентами для локальных
